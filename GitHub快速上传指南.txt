【第一步】注册GitHub账号（如果还没有）
================================================================================

1. 访问：https://github.com

2. 点击 "Sign up" 注册

3. 填写信息：
   - Email：您的邮箱
   - Password：设置密码
   - Username：用户名（全球唯一）

4. 验证邮箱

5. 注册完成！


【第二步】创建新仓库
================================================================================

1. 登录后，点击右上角头像旁边的 "+" 号

2. 选择 "New repository"

3. 填写仓库信息：

   Repository name（仓库名）:
   ┌────────────────────────────────┐
   │ CGB-SMS-Forwarder              │  ← 随便起个名字
   └────────────────────────────────┘

   Description（描述，可选）:
   ┌────────────────────────────────┐
   │ 短信验证码转发系统              │
   └────────────────────────────────┘

   可见性：
   ☑ Public（公开）  ← 选这个！免费使用Actions
   ☐ Private（私有）

   初始化选项：
   ☑ Add a README file  ← 勾选这个

4. 点击绿色按钮 "Create repository"

5. 创建成功！会自动跳转到仓库页面


【第三步】准备上传的文件
================================================================================

在上传前，请确保以下文件已准备好：

必须上传的文件/文件夹：
✅ android_client/（整个文件夹）
   - 包含所有Android源码
   
✅ .github/（整个文件夹）
   - 包含工作流配置文件
   - .github/workflows/build-android.yml
   
✅ server.py
✅ config.py  
✅ requirements.txt
✅ README.md
✅ .gitignore

可选上传：
📄 使用说明.txt
📄 开始使用.txt
📄 其他文档.txt

⚠️ 不要上传的文件：
❌ verification_codes.txt（包含验证码，隐私文件）
❌ __pycache__/（Python缓存）
❌ .idea/（IDE配置）
❌ *.pyc（Python编译文件）
❌ build/（编译输出）
❌ .gradle/（Gradle缓存）


【第四步】上传文件到GitHub
================================================================================

方法1：拖拽上传（最简单）⭐⭐⭐⭐⭐
----------------------------------

1. 在仓库页面，点击 "Add file" → "Upload files"

2. 打开文件资源管理器，导航到：
   D:\pythonjob\CGB自动认证\

3. 选中要上传的文件和文件夹：
   - 按住Ctrl，点击选择多个文件/文件夹
   - 或者直接全选（Ctrl+A），但要排除不需要的

4. 将选中的文件拖拽到GitHub页面的上传区域
   （显示"Drag files here to add them to your repository"）

5. 等待文件上传完成（会显示上传进度）

6. 在底部 "Commit changes" 区域：
   
   提交信息（必填）:
   ┌────────────────────────────────┐
   │ 上传短信转发系统代码            │
   └────────────────────────────────┘

   详细描述（可选）:
   ┌────────────────────────────────┐
   │ - 添加电脑服务端程序           │
   │ - 添加Android客户端程序        │
   │ - 添加GitHub Actions配置       │
   └────────────────────────────────┘

7. 点击绿色按钮 "Commit changes"

8. 等待上传处理完成


方法2：分批上传（如果一次上传失败）
----------------------------------

如果文件太多或太大，可以分批上传：

第一批：上传.github文件夹
1. Add file → Upload files
2. 只拖入 .github 文件夹
3. Commit changes

第二批：上传android_client文件夹
1. Add file → Upload files
2. 只拖入 android_client 文件夹  
3. Commit changes

第三批：上传其他文件
1. Add file → Upload files
2. 拖入剩余的.py、.txt、.md文件
3. Commit changes

上传完成后，仓库页面应该显示：

关键检查：

✅ android_client/ 文件夹完整
✅ 没有上传 verification_codes.txt


【第六步】查看自动编译
================================================================================

上传完成后，GitHub Actions会自动开始编译！

1. 点击仓库页面顶部的 "Actions" 标签

2. 看到工作流列表，最新的一个会显示：
   🟡 黄色圆圈转圈 = 正在编译
   ✅ 绿色对勾 = 编译成功
   ❌ 红色叉号 = 编译失败

3. 点击进入工作流，查看详细进度

4. 等待3-5分钟，编译完成


【第七步】下载APK文件
================================================================================

编译成功后：

1. 在工作流详情页面，滚动到底部

2. 找到 "Artifacts" 区域

3. 会看到：
   📦 app-debug    XX MB    Download

4. 点击 "Download" 下载ZIP文件

5. 解压ZIP，得到：app-debug.apk

6. 将APK传输到手机：
   - 方法1：USB连接电脑，复制到手机
   - 方法2：通过QQ/微信发送到手机
   - 方法3：上传到网盘，手机下载

7. 在手机上点击APK文件安装


【第八步】手动触发编译（可选）
================================================================================

如果您想随时重新编译：

1. Actions → 左侧选择 "编译Android应用"

2. 右上角点击 "Run workflow" 下拉按钮

3. 保持默认选项，点击绿色 "Run workflow"

4. 刷新页面，看到新的工作流开始运行

5. 等待完成，下载新的APK


【常见问题】
================================================================================

Q1: 上传时提示文件太大？
A: GitHub单个文件限制100MB，仓库限制1GB。
   - android_client不应该超过限制
   - 如果超过，检查是否包含了build/目录（不应上传）

Q2: 上传后没有自动编译？
A: 检查：
   - .github/workflows/build-android.yml 是否存在
   - 路径是否正确（必须是 .github/workflows/ 下）
   - 手动触发试试

Q3: 编译失败？
A: 点击失败的工作流，查看日志。常见原因：
   - gradle文件缺失
   - 依赖下载失败（重新运行即可）

Q4: 下载APK很慢？
A: GitHub服务器在国外，可能较慢。使用加速工具或让朋友帮忙。

Q5: 我想修改代码后重新编译？
A: 
   方法1：在GitHub网页上直接编辑文件，保存即触发编译
   方法2：重新上传修改后的文件
   方法3：学习使用Git命令行

Q6: 如何删除仓库？
A: Settings → 滚动到底部 → Delete this repository
