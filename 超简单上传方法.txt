【方法：使用GitHub网页版上传】
================================================================================

完全不需要：
❌ 安装Git
❌ 输入命令
❌ 记住密码
❌ 生成Token

只需要：
✅ 一个浏览器
✅ 能上网
✅ 会拖拽文件

时间：5-10分钟搞定！


【详细步骤】
================================================================================

步骤1：登录GitHub
----------------
1. 打开浏览器
2. 访问：https://github.com/xrq3388/smsapp
3. 如果未登录，点击右上角 "Sign in" 登录
   用户名：xrq3388
   密码：您的密码


步骤2：上传文件
--------------
1. 在仓库页面，点击右上方的 "Add file" 按钮
2. 选择 "Upload files"
3. 会看到一个上传区域（显示"Drag files here..."）


步骤3：准备要上传的文件
---------------------
打开文件资源管理器，导航到：
D:\pythonjob\CGB自动认证\

选择以下文件夹和文件：

✅ 必须上传：
  📁 .github（整个文件夹）
  📁 android_client（整个文件夹）
  📄 server.py
  📄 config.py
  📄 requirements.txt
  📄 README.md
  📄 .gitignore
  📄 使用说明.txt
  📄 开始使用.txt
  📄 其他.txt文件（可选）

❌ 不要上传：
  📄 verification_codes.txt（包含您的验证码，隐私！）
  📁 __pycache__（如果有）


步骤4：拖拽上传
--------------
1. 选中上面的文件和文件夹（按住Ctrl多选）
2. 直接拖拽到GitHub网页的上传区域
3. 看到文件列表出现，等待上传进度条
4. 大文件可能需要几分钟，耐心等待


步骤5：提交更改
--------------
上传完成后，页面底部会显示：

Commit changes
┌─────────────────────────────────────────┐
│ 上传短信转发系统代码                     │  ← 输入提交说明
├─────────────────────────────────────────┤
│ - 添加电脑服务端程序                     │  ← 可选的详细描述
│ - 添加Android客户端                      │
│ - 添加GitHub Actions配置                 │
└─────────────────────────────────────────┘

点击绿色按钮 "Commit changes"


步骤6：等待处理
--------------
GitHub会处理上传的文件，可能需要10-30秒。
完成后会自动跳转到仓库主页。


步骤7：查看自动编译
-----------------
1. 在仓库主页，点击顶部的 "Actions" 标签
2. 会看到一个工作流正在运行（黄色圆圈转圈）
3. 这是自动编译Android应用的过程


步骤8：等待编译完成
-----------------
编译通常需要3-5分钟。

状态指示：
🟡 黄色圆圈 = 正在编译（耐心等待）
✅ 绿色对勾 = 编译成功（可以下载了）
❌ 红色叉号 = 编译失败（点击查看日志）


步骤9：下载APK文件
-----------------
编译成功后：

1. 点击绿色对勾的工作流进入详情页
2. 滚动到页面底部
3. 找到 "Artifacts" 区域
4. 看到：
   📦 app-debug    XX MB    [Download]
5. 点击 "Download" 下载
6. 会下载一个ZIP文件


步骤10：解压和安装
-----------------
1. 下载的ZIP文件解压
2. 得到：app-debug.apk
3. 传输到手机：
   - USB连接电脑复制
   - 或用QQ/微信发送到手机
4. 在手机上点击APK安装
5. 可能需要允许"安装未知应用"


【完成！】
================================================================================

✅ 代码已上传到GitHub
✅ 自动编译完成
✅ APK已下载
✅ 可以在手机上使用了！


【重要提示】
================================================================================

上传时注意：

1. ⚠️ 不要上传 verification_codes.txt
   这个文件包含您的验证码历史，是隐私信息！

2. ✅ 必须上传 .github 文件夹
   这个文件夹包含自动编译的配置，不上传就不会自动编译

3. ✅ 必须上传完整的 android_client 文件夹
   这是Android应用的源代码

4. 🌐 如果文件很多，可以分批上传
   第一批：.github 文件夹
   第二批：android_client 文件夹
   第三批：其他文件


【如果上传失败】
================================================================================

问题1：文件太大无法上传
---------------------
GitHub单个文件限制100MB。

解决：
- 检查是否包含了不该上传的文件
- android_client/build/ 不应该存在（编译产物）
- android_client/.gradle/ 不应该存在（缓存）


问题2：网络超时
--------------
解决：
- 分批上传，每次传少量文件
- 换个网络环境
- 使用Git命令行（更稳定）


问题3：没有自动编译
------------------
解决：
- 检查 .github/workflows/build-android.yml 是否上传成功
- 路径必须完全正确（包括 .github 前面的点）
